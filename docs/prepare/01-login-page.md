# Poprako W 登录页与注册页手册

## 页面定位

`/login` 是 Poprako W 的统一认证入口。

它承担两类任务：

1. 已有账号用户登录系统。
2. 持有邀请码的新用户注册账号并立即进入系统。

登录或注册成功后，页面会保存访问令牌，并跳转到 `/comic-playground`。

## 页面结构

页面主体是居中的登录卡片，内部只有一组核心区域：

1. 品牌标题 `PopRaKo W`。
2. 模式切换栏：`登录`、`注册`。
3. 表单输入区。
4. 提交按钮。

注册模式相较登录模式会额外出现两个字段：

1. 昵称。
2. 邀请码。

## 主要功能

### 1. 登录

用户填写：

1. QQ 号。
2. 密码。

提交后，前端会调用登录接口。成功时：

1. 写入 `accessToken`。
2. 清空本地缓存的登录态，迫使后续页面重新拉取用户信息。
3. 显示“登录成功”。
4. 跳转到 `/comic-playground`。

### 2. 注册

用户填写：

1. QQ 号。
2. 密码。
3. 昵称。
4. 邀请码。

邀请码来自成员页中的邀请功能。注册成功后，行为与登录一致，也会直接进入系统。

### 3. 表单基础校验

提交前页面会做最基本的必填校验：

1. QQ 号不能为空。
2. 密码不能为空。
3. 注册时昵称不能为空。
4. 注册时邀请码不能为空。

若不满足条件，会直接弹出错误提示，不会发起请求。

## 典型使用流程

### 场景一：已有用户登录

1. 打开 `/login`。
2. 保持在“登录”标签。
3. 输入 QQ 号和密码。
4. 点击“登录”。
5. 成功后自动进入漫画主页面。

### 场景二：新成员注册

1. 从团队管理员处获得邀请码。
2. 打开 `/login`。
3. 切换到“注册”。
4. 填写 QQ 号、密码、昵称、邀请码。
5. 点击“注册”。
6. 成功后直接进入系统。

## 反馈与状态

1. 提交过程中按钮会变为“处理中…”。
2. 请求失败时会弹出错误提示。
3. 前端会把异常打印到控制台，便于排查。

## 白杨子可如何描述本页

适合对白杨子描述为：

“这是 Poprako W 的账号入口页，支持登录和基于邀请码的注册。注册成功后用户会立刻成为某个汉化组成员，并进入漫画操作页。”

## 与其他页面的关系

1. 受保护页面在发现未登录时，会跳回 `/login`。
2. 登录成功后默认进入 `/comic-playground`，而不是 `/workspace`。
3. 如果路由加载出错，可能会显示错误页。
