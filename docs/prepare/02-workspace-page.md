# Poprako W 工作区页面手册

## 页面定位

`/workspace` 是用户的个人工作台。

它不是按“全团队所有漫画”组织，而是按“当前用户手上相关任务”组织，强调三件事：

1. 我最近要处理哪些漫画。
2. 当前汉化组有什么公告。
3. 组内最近在说什么。

## 页面结构

页面自上而下主要由四部分组成：

1. 欢迎区。
2. 当前公告。
3. 任务列表。
4. 留言板。

桌面端中，任务列表和留言板左右分栏。
移动端中，任务列表和留言板切换为两个横向滑动标签页。

## 功能详解

### 1. 欢迎区

页面顶部会展示当前登录用户的昵称，形成“欢迎回来”的工作台感知。

### 2. 当前公告

公告区域基于当前选中的汉化组加载，默认最多显示最近 3 条。

用户可以：

1. 浏览公告标题、摘要和发布时间。
2. 点击某条公告，打开右侧详情抽屉查看完整内容。

如果当前成员拥有管理权限，还可以：

1. 点击右上角加号按钮。
2. 打开公告创建弹窗。
3. 填写标题和正文。
4. 发布后刷新公告列表。

### 3. 任务列表

任务列表展示的是“当前用户参与过或被分配到的漫画集合”，而不是团队内所有漫画。

它具备以下特征：

1. 使用无限滚动加载。
2. 以卡片形式展示漫画。
3. 点击漫画卡片会打开漫画详情弹窗。

这个列表适合白杨子理解为“我的任务入口集合”。

### 4. 留言板

留言板是团队级别的轻量交流区。

用户可以：

1. 查看最近留言。
2. 输入文字后发送。
3. 使用 `Enter` 发送，`Shift + Enter` 换行。

发送成功后，页面会重新加载留言，确保列表同步。

## 漫画详情弹窗

工作区中点击漫画后，会进入一个高权重的漫画详情弹窗。它是本页最重要的深层操作入口。

在这个弹窗中，用户可执行的操作受身份和章节状态影响，常见包括：

1. 查看漫画基本信息。
2. 切换章节、加载更多章节。
3. 新建章节。
4. 删除章节。
5. 查看章节页面缩略图。
6. 上传原图页面。
7. 清空当前章节全部页面后重新上传。
8. 对单页执行重传。
9. 导入章节数据。
10. 导出章节数据。
11. 导出章节 LP 数据。
12. 指派各流程成员。
13. 自己加入某个分工角色。
14. 自己退出某个分工角色。
15. 推进章节工作流。
16. 进入翻译器。
17. 只读打开翻译器。
18. 在管理员权限下删除整部漫画。

## 典型使用流程

### 场景一：查看今天要处理的漫画

1. 进入 `/workspace`。
2. 在任务列表中浏览漫画卡片。
3. 点击目标漫画。
4. 在详情弹窗中确认章节和页面。
5. 点击具体页面，进入翻译器。

### 场景二：管理员发布组内公告

1. 进入 `/workspace`。
2. 在公告区点击加号。
3. 填写标题和内容。
4. 发布后返回公告卡片列表。

### 场景三：在留言板同步进度

1. 进入 `/workspace`。
2. 在留言输入框中输入进度说明。
3. 点击发送图标或按 `Enter`。
4. 留言刷新后出现在列表底部。

## 页面限制与规则

1. 公告区和留言板都依赖当前选中的汉化组；没有选中的组时，这些区域不会正常工作。
2. 发布公告需要管理员权限。
3. 漫画详情中的上传、分工、流程推进、删除漫画等操作都受角色限制。
4. 移动端只有在存在团队上下文时，才会显示“任务列表/留言板”的切换式内容区。

## 与其他页面的关系

1. `/` 会先跳到 `/workspace`。
2. 从本页打开漫画后，会进入漫画详情弹窗。
3. 从漫画详情点击页面，会跳转到 `/translator/:chapterId/:pageId`。
4. 翻译器退出时，如果带有 `returnTo=/workspace`，就会回到本页，并恢复漫画与章节定位。
5. 左侧边栏或移动端底部导航可跳往漫画页、成员页、消息页、设置页。

## 白杨子可如何描述本页

“工作区是个人视角的任务总览页，聚合了当前用户相关漫画、组内公告和轻量留言板，适合作为日常进入后的第一工作面板。”
