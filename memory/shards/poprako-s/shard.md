poprako-s 是白杨汉化组的项目管理后端，使用 Go 语言开发，提供 RESTful API。以下是其完整功能范畴：

【用户系统】
- 注册、登录（JWT 认证）
- 获取用户信息、更新个人资料
- 头像上传（通过 OSS 对象存储签名上传）
- 最后活跃时间追踪

【团队管理】
- 创建团队、获取团队详情、更新团队信息
- 团队列表（全部/按用户筛选）
- 团队头像管理（预留签名上传 URL + 确认上传）

【成员管理】
- 添加成员到团队、按团队列出成员、列出用户所属的所有团队
- 更新成员角色掩码（角色权限基于位掩码）
- 删除成员
- 通过邀请码加入团队

【团队邀请系统】
- 创建邀请、列出团队下的邀请、更新邀请角色掩码、删除邀请

【作品集管理（Workset）】
- 创建作品集、列出团队下的作品集、更新作品集（名称/简介）、删除作品集

【漫画管理】
- 创建漫画、列出作品集下的漫画、获取漫画详情
- 更新漫画信息、删除漫画
- 封面图管理（预留签名上传 URL + 确认上传）

【章节管理】
- 创建章节（自动置顶为当前章节）、列出漫画下的章节、获取章节详情
- 获取漫画的当前置顶章节（pinned chapter）
- 更新章节字段、删除章节
- 加入章节任务分配

【章节导入导出】
- 导出章节翻译数据为 JSON 格式
- 导出为 LabelPlus 文本格式（供外部工具使用）
- 从支持的格式导入章节数据

【页面管理】
- 列出章节下的页面

【翻译单元管理（Unit）】
- 列出页面下的所有翻译单元（文本框）
- 保存页面翻译单元变更（基于 diff 同步计数）

【任务分配管理（Assignment）】
- 列出章节下的任务分配、列出用户的所有任务分配
- 增删改任务分配角色（基于角色掩码的 put 语义 upsert）
- 删除任务分配

【任务分配邀请】
- 创建章节任务分配邀请、列出章节下的邀请
- 通过邀请码加入章节任务

【工作流追踪】
- 每个章节独立追踪 5 个阶段：上传（图源）→ 翻译 → 校对 → 嵌字 → 监修 → 发布
- 每个阶段记录开始时间和完成时间（部分阶段只有完成时间）
- 事件驱动：下一阶段触发时通知相关人员

【系统邮件】
- 列出用户的未读系统邮件（分页）
- 标记邮件为已读

【公告系统】
- 创建团队公告、列出团队公告

【评论系统】
- 团队留言板：创建评论、列出评论

【事件系统】
- 内部事件总线，支持订阅/发布模式
- 流程进度变更时自动发送系统通知（通知邀请者、通知下一阶段责任人）

【对象存储（OSS）】
- 支持头像、漫画封面、章节页面图片的上传
- 后台 worker 处理待处理的 OSS 创建/删除操作
